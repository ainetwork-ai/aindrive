# Permit2 settlement via x402 v2 (Phase 2b) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** EIP-3009 미지원 ERC-20(FANCO 등)을 x402 v2 permit2 경로로 실온체인 정산 가능하게 한다.

**Architecture:** 결제 스택을 x402 v1(`x402-fetch@1.2.0`)에서 v2(`@x402/*@2.14.x`) 코어 프리미티브로 마이그레이션하되 라우트/게이트의 도메인 로직 구조는 유지. 토큰별 `transferMethod`를 정책에 명시 저장하고, permit2 토큰은 클라이언트 2단계(approve→pay) UX + 서버 412 폴백으로 처리. Spec: `docs/superpowers/specs/2026-06-11-permit2-x402v2-design.md`.

**Tech Stack:** Next.js 15, vitest, wagmi/viem, `@x402/core` `@x402/evm` `@x402/fetch` `@coinbase/x402`.

```ts
export type TransferMethod = "eip3009" | "permit2";
export type PaymentToken = {
  symbol: string; chain: string; asset: string;
  name: string | null; version: string | null; decimals: number;
  transferMethod: TransferMethod;
};
// isPaymentToken: transferMethod는 누락 허용("eip3009"|"permit2"만 유효),
// normalize 단계에서 레거시 추론: name && version ? eip3009 : permit2.
// parseTokenPolicy가 normalize까지 수행해 반환 토큰은 항상 method 보유.
export function isX402Settleable(t: Pick<PaymentToken, "name" | "version" | "asset" | "transferMethod">): boolean {
  if (t.transferMethod === "permit2") return !!t.asset;
  return !!(t.name && t.version && t.asset);
}
const CAIP2_BY_CHAIN: Record<string, string> = { base: "eip155:8453", "base-sepolia": "eip155:84532" };
export function toCaip2Network(chain: string): string {
  const id = CAIP2_BY_CHAIN[chain];
  if (!id) throw new Error(`unknown payment chain: ${chain}`);
  return id;
}
```

프리셋: USDC 양 네트워크에 `transferMethod: "eip3009"`, FANCO에 `"permit2"` + 주석에서 "Phase 2b 대기" 문구를 "permit2로 정산" 으로 교체.

```ts
import { wrapFetchWithPayment, x402Client } from "@x402/fetch";
import { ExactEvmScheme, createPermit2ApprovalTx, getPermit2AllowanceReadParams } from "@x402/evm/exact/client";

// Mirrors @x402/paywall browserAdapter (not publicly exported): wagmi
// WalletClient → ClientEvmSigner. 결제 서명은 signTypedData만 필요.
function toSigner(wc: WalletClient) {
  return {
    address: wc.account!.address,
    signTypedData: (m: any) => wc.signTypedData({ account: wc.account!, domain: m.domain, types: m.types, primaryType: m.primaryType, message: m.message }),
  };
}

const client = new x402Client()
  .register("eip155:*", new ExactEvmScheme(toSigner(walletClient)))
  // v1 maxValue 가드 대체: 표시가보다 큰 금액 requirement는 결제 후보에서 제외.
  .registerPolicy((_v, reqs) => reqs.filter((r) => BigInt(r.amount) <= BigInt(requirement.amount)));
const fetchWithPay = wrapFetchWithPayment(globalThis.fetch, client);
const res = await fetchWithPay(`/api/s/${token}`);
if (res.status === 412) { setNeedsApproval(true); return; }  // allowance race 폴백
```

permit2 사전 체크 + 승인 단계: requirement.extra?.assetTransferMethod === "permit2"이면 `usePublicClient().readContract(getPermit2AllowanceReadParams({ tokenAddress, ownerAddress }))` → 부족 시 paywall 버튼이 "Approve {symbol} (one-time)"로: `walletClient.sendTransaction(createPermit2ApprovalTx(asset))` → `publicClient.waitForTransactionReceipt` → allowance 재확인 → Pay 버튼 활성화. UI에 2단계임을 알리는 caption 한 줄.
